# 信息源清单（Source manifest）

每次根据本 skill **生成或重大更新**攻略 HTML 后，Agent **必须**另写一份清单文件，把本次用到的信息源集中记录下来，便于审计、复现与后续改稿。

---

## 文件怎么命名、放哪

- **默认**：与输出 HTML **同目录、同 basename**，扩展名 `.sources.md`  
  - 例：`docs/trips/alps.html` → `docs/trips/alps.sources.md`  
- 若用户指定目录但不便同 basename，用 **`<basename>.sources.md`** 或用户给定路径；在对话里说明最终路径。

---

## 清单里要写什么

按实际情况勾选填写，**没有则写「无」**。

### 1. 元信息

- 生成时间（建议 ISO 8601，如 `2026-04-02T12:00:00+08:00`）
- 对应 **输出 HTML 路径**（仓库内相对路径）
- 使用的 **skill 版本提示**（可选：写 `doc-to-travel-guide-web` + 仓库路径）

### 2. 结构化真源

- **YAML / CSV / 表格文件**路径（若存在）
- **Notion**：页面标题 + URL（若用户曾提供）
- **Confluence**：页面标题 + URL（若用 access-confluence 等读过）
- **聊天内粘贴**：用**简短描述**标注（如「用户粘贴 2026-04-24～04-26 三日列表」），**勿**把完整确认号、护照级隐私写入清单；若只有粘贴无文件，写清楚「无持久化源文件，仅对话」

### 3. 地图与外部链接

用表格或列表列出进入 HTML 的 **Google Maps / 官网 / 订票** 链接：

| 标签（如 D1 主路线） | 来源 | 说明 |
|----------------------|------|------|
| … | `user` / `agent-generated` / `template` | 用户粘贴的完整 URL / 由 `route_spec` 拼接 / 来自范例 |

对 **Agent 拼接**的 `maps/dir`，写清 `origin` / `destination` / `waypoints` 摘要（不必重复整串 URL 也可只写「见 HTML 内 href」+ 摘要）。

### 4. 图片

- 外链图：每个 **稳定 URL** 一行（或写「同 italy-trip 范例 Unsplash 集」）
- 本地图：相对路径

### 5. 推断与待核对（重要）

凡 **非用户原文明确给出** 而由 Agent **补全或猜测**的字段（如默认 `overnight_date`、从散文拆出来的时间表），在此 **逐条列出**，提醒用户核对。

## 隐私

- 清单用于协作与追溯，**避免**完整预订确认号、证件号；可写「确认号见用户私有 YAML / Notion」。
