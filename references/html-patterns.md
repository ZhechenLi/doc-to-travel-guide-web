# 攻略单页 HTML — 结构与风格

## 视觉原则

**目标**：看起来像一个人认真做的旅行手册，不像 AI 批量生成的模板。

### 要

- 干净的排版，充足的留白
- 克制的配色——一个主色 + 一个强调色足够，从行程目的地取灵感（意大利可以是暖赭 + 深蓝，日本可以是墨灰 + 朱红）
- 信息密度高但不拥挤：该折叠的折叠，该省略的省略
- 字体：正文用系统字体栈（`-apple-system, 'Segoe UI', sans-serif`），不额外引入花哨字体

### 不要

- 渐变彩虹背景、霓虹色阴影
- Emoji 堆砌当标题装饰（一两个有意义的 emoji 可以，但不要每行都加）
- AI 套话：「祝你旅途愉快！」「这将是一次难忘的旅程～」「以下是为您精心整理的行程」
- 空洞的 hero 大图 + 半透明遮罩（除非用户提供了真实照片）
- 无意义的动画

---

## 页面骨架

- 单文件：`DOCTYPE`、`<meta viewport>`、`<title>` 含路线关键词
- **页头**：行程名、日期区间、途经城市列表——简短，一目了然
- **按日时间线**：核心内容区
- **页脚**：配图版权 / 免责声明（如有）

## 地图规则

### 核心：按城市拆图，跨城不混

Agent 必须先识别每天的 `day_type`（见 content-taxonomy.md §1b），再决定地图怎么画。**不同类型的地图完全不同。**

### 按 day_type 选择地图

| `day_type` | 地图 | 说明 |
|------------|------|------|
| **`city`** | 📍 **点位地图** | Leaflet 标注景点/餐厅/酒店，不画路线连线。`fitBounds` 到点位范围。视觉重心是「今天去哪几个地方」 |
| **`drive`** | 🛣️ **驾车路线图** | Google Maps 嵌入或 Leaflet polyline，画出完整路线（起点→途经点→终点）。视觉重心是「今天走哪条路」。途经的景点/停靠点标注在路线上 |
| **`transfer`** | ❌ **不画地图** | 火车/飞机移动用文字卡片表达（交通方式、时刻、时长）。到达后如果有少量活动，可在目的地城市的地图中体现 |
| **`mixed`** | 🔀 **拆开处理** | 交通段（火车/飞机）用文字卡片，不画地图；城市段按 `city` 处理画点位图；自驾段按 `drive` 处理画路线图 |

### city 地图 vs drive 地图的核心区别

```
City walk 地图                    自驾地图
┌─────────────────┐              ┌─────────────────┐
│                  │              │                  │
│   📍 浅草寺      │              │  A ──→ B ──→ C   │
│                  │              │  │         │     │
│      📍 上野公园  │              │  ↓         ↓     │
│                  │              │  D ──→ E ──→ F   │
│  📍 六歌仙       │              │                  │
│                  │              │  标注停靠点在路线上 │
│  散点标注，无连线  │              │  路线是主角        │
└─────────────────┘              └─────────────────┘
```

### 同城市跨天

同一个城市连续多天（如米兰 Day 1-2）：

- 如果都是 `city` 类型：**可以共享一张地图**标注所有点位，或每天一张——视点位密度决定
- 如果某天是 `drive`（如从米兰出发自驾）：该天单独一张路线图

### 城市切换分隔

在城市变化时插入分隔带：

```html
<section class="city-divider">
  <h2>威尼斯</h2>
  <p class="city-subtitle">04.26 — 半日水城</p>
</section>
```

一个城市名 + 日期/一句话。简洁、无装饰。

### 地图实现

- **Leaflet**（推荐用于 `city` 点位图）：轻量、免费瓦片。每个城市一个 `L.map` 实例
- **Google Maps 嵌入**（推荐用于 `drive` 路线图）：用户提供的 `routes[].url` 或 Agent 拼接的 `maps/dir` 链接，`<details>` 折叠 + 懒加载。驾车路线的 iframe 显示效果比 Leaflet 好
- **Google 导航按钮**：始终保留，无论地图类型

## 单日卡片内容顺序

1. **日期 + 标题**：`Day 3 · 04.26 周日 · 秋叶原 + 台场`
2. **交通**（若有城际移动）：一行文字，不是地图
3. **当日地图**（可选）：该城市内的点位
4. **时间表**：`<ul class="schedule">`
5. **Google 导航按钮**
6. **路线嵌入**（可选折叠）
7. **餐厅预订卡片**
8. **美食推荐**（与餐厅视觉区分：轻量列表或侧栏）
9. **景点折叠**
10. **住宿卡片**（当日入住的酒店）
11. **备注**

## CSS 变量（示例）

```css
:root {
  --bg: #fafaf8;
  --text: #2c2c2c;
  --text-secondary: #6b6b6b;
  --accent: #c45a3c;       /* 从目的地取灵感 */
  --card-bg: #ffffff;
  --border: #e5e5e5;
}
```

不需要每个城市一个主题色。一套配色贯穿全页，干净统一。

## 性能与交互

- Google iframe：`<details>` + `toggle` 监听设 `src`，**不首屏加载**
- 图片：`loading="lazy"`；外链先校验 HTTP 200
- 动画：最多用 `opacity` 淡入，不要弹跳、旋转、滑入

## 打印

- `@media print`：隐藏折叠按钮、固定 `day-card` 不透明
- 地图打印为静态截图或隐藏（Leaflet 打印效果差）
